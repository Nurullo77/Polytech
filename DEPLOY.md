# Как задеплоить сайт на GitHub

---

## Вариант «просто скопировать файлы» (без Git в терминале)

Если у тебя уже есть репозиторий на GitHub и хочешь просто закинуть туда файлы:

### 1. Открой свой репозиторий на github.com

### 2. Загрузи файлы проекта

- Нажми **"Add file"** → **"Upload files"**.
- Открой на компьютере папку **`сайт школы`**.
- Перетащи в окно браузера **все файлы и папки**, кроме папки **`node_modules`** (её не загружай).

Нужно загрузить:
- папка **`.github`** (внутри неё `workflows/deploy.yml`);
- папка **`src`** (все файлы внутри);
- файлы: **`index.html`**, **`package.json`**, **`package-lock.json`**, **`vite.config.js`**, **`tailwind.config.js`**, **`postcss.config.js`**, **`.gitignore`**, **`README.md`**, **`DEPLOY.md`** (если есть).

Если папки **`.github`** в проводнике не видно — в проводнике Windows: **Вид** → включи **«Скрытые элементы»**, тогда появится папка `.github`.

### 3. Укажи имя репозитория в проекте

Открой в проекте файл **`vite.config.js`**. В строке **`base:`** должно быть имя **твоего** репозитория. Например, если репозиторий называется **`my-school-site`**, должно быть:

```js
base: '/my-school-site/',
```

Исправь, сохрани файл и снова загрузи этот файл в репозиторий (можно через **"Add file"** → **"Upload files"** и заменить старый).

### 4. Нажми **"Commit changes"** (зелёная кнопка внизу)

### 5. Включи GitHub Pages

В репозитории: **Settings** → слева **Pages** → в блоке **Source** выбери **GitHub Actions**.

Через 1–2 минуты зайди во вкладку **Actions** — должен запуститься workflow и собрать сайт. Когда он станет зелёным, сайт откроется по адресу:

**https://ТВОЙ_ЛОГИН.github.io/ИМЯ_ТВОЕГО_РЕПОЗИТОРИЯ/**

## Вариант через Git в терминале

## Часть 1: Загрузить код на GitHub

### Шаг 1. Установи Git (если ещё нет)
Скачай с https://git-scm.com/download/win и установи.

### Шаг 2. Открой терминал в папке проекта
Например: **PowerShell** или **CMD** в Cursor (Terminal → New Terminal).

Перейди в папку:
```bash
cd "c:\Users\HP\Desktop\сайт школы"
```

### Шаг 3. Инициализируй репозиторий и сделай первый коммит
Выполни по очереди:

```bash
git init
git add .
git commit -m "Первый коммит: сайт Polytech School"
```

### Шаг 4. Создай репозиторий на GitHub
1. Зайди на https://github.com и войди в аккаунт.
2. Нажми **"+"** → **"New repository"**.
3. Название репозитория: **polytech-school** (латиницей, без пробелов).  
   Если возьмёшь другое имя — потом в проекте в файле `vite.config.js` измени `base: '/твоё-имя-репо/'`.
4. Оставь **Public**, не добавляй README (он уже есть в проекте).
5. Нажми **"Create repository"**.

### Шаг 5. Подключи удалённый репозиторий и отправь код
На странице нового репозитория GitHub скопируй URL (например `https://github.com/ТВОЙ_ЛОГИН/polytech-school.git`).

В терминале выполни (подставь свой URL):

```bash
git remote add origin https://github.com/ТВОЙ_ЛОГИН/polytech-school.git
git branch -M main
git push -u origin main
```

Готово: код сайта теперь на GitHub.

## Часть 2: Опубликовать сайт (GitHub Pages)

Чтобы сайт открывался по ссылке вида `https://ТВОЙ_ЛОГИН.github.io/polytech-school/`:

### Шаг 1. Узнай точное имя репозитория
Например: **polytech-school**.

### Шаг 2. Включи GitHub Pages
1. Открой свой репозиторий на GitHub.
2. **Settings** → слева **Pages**.
3. В **Source** выбери **"GitHub Actions"**.

### Шаг 3. Добавь workflow для сборки и деплоя
В проекте уже есть файл `.github/workflows/deploy.yml` — он собирает сайт и публикует его на GitHub Pages. После того как ты сделаешь **push** (см. Часть 1), workflow запустится сам.

Если репозиторий называется **не** `polytech-school`, открой файл  
`vite.config.js` и в `base` укажи своё имя репозитория (с слэшами), например:
`base: '/мое-название-репо/'`.

### Шаг 4. Подожди 1–2 минуты
После **push** зайди в репозиторий → вкладка **Actions**. Когда workflow станет зелёным, сайт будет доступен по адресу:

**https://ТВОЙ_ЛОГИН.github.io/ИМЯ_РЕПОЗИТОРИЯ/**

(Имя репозитория — то, что ты ввёл при создании репозитория, например `polytech-school`.)
